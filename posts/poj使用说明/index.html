1. POJ的提示说明与部分错误示范

  1. Waiting：你的程序刚刚提交，正在就包括评测。
  2. Accepted：你的程序是正确的
  3. Wrong Answer：运行结果和预设的样例结果不符
  4. Time Limit Exceed（Memory Limit Exceed）：程序的运行时间（内存占用）超出了题目的要求，在这门课程中很少会出现这两个错误
  5. Output Limit Exceed：程序输出太多内容，超过了POJ的限制，这个时候一般是程序中有死循环导致的
  6. Runtime Error: 程序运行中出错，异常终止了，下面是从同学们的提交中出现的典型的此类错误以及原因：
# 作业1 苹果与虫子
# 这种是最常见的情况，题目要求每次输入一行
# 但是在代码里每次输入了三行，在这种情况下，
# openjudge的系统会直接终止评测程序的运行
n,x,y=input(),input(),input()
n=int(n)
x=int(x)
y=int(y)
last=int(n-y/x)
print(last)

# 作业1 比大小
# 这个题目题干要求输入的数字可以是整数也可以是小数
# 在这位同学的提交中，默认输入时整数，当输入为小数
# 时，python运行时就会出错退出引发Runtime Error
# 同时，这位同学第一题的代码语法正确但属于高级用法
# 不推荐各位同学使用
n: int = int(input(""))
m = int(input(""))
if n>m:
    print(True)
else:
    print("False")

# 作业2 计算器
# 这位同学的程序中，出现了除以0的操作，这会引起ZeroDivisionError中断程序
lst = "+,-,*,/"
n = input()
a,c,b = n.split()
a = int(a)
b = int(b)
if c not in lst:
    print("Invalid operator!")
else:
    if c == "+":
        print(a+b)
    if c == "-":
        print(a-b)
    if c == "*":
        print(a*b)
    if c == "/":
        ans = float(a/b)
        print("{:.2f}".format(ans))
        if b == 0:
            print("Divided by zero!")
